# Google Cloud Run 部署指南

本指南将帮助您将 Lighthouse 监控平台部署到 Google Cloud Run。

## 1. 前提条件

1.  拥有一个 Google Cloud Platform (GCP) 账号。
2.  创建一个新的 GCP 项目（例如 `lighthouse-hq`）。
3.  安装并配置 [gcloud CLI](https://cloud.google.com/sdk/docs/install)。
4.  启用以下 API：
    *   Cloud Run API
    *   Artifact Registry API

```bash
# 登录 gcloud
gcloud auth login

# 设置当前项目
gcloud config set project [YOUR_PROJECT_ID]

# 启用 API
gcloud services enable run.googleapis.com artifactregistry.googleapis.com
```

## 2. 创建 Artifact Registry 仓库

我们需要一个地方来存储 Docker 镜像。

```bash
# 创建一个名为 "docker-repo" 的仓库，位于 "us-central1" (或其他您喜欢的区域)
gcloud artifacts repositories create docker-repo \
    --repository-format=docker \
    --location=us-central1 \
    --description="Docker repository for LighthouseHQ"
```

## 3. 构建并推送镜像

在项目根目录下运行以下命令：

```bash
# 设置变量
export PROJECT_ID=$(gcloud config get-value project)
export REGION="us-central1"
export REPO="docker-repo"
export IMAGE="lighthouse-hq"
export TAG="latest"

# 构建镜像 (使用 Cloud Build，无需本地安装 Docker)
gcloud builds submit --tag $REGION-docker.pkg.dev/$PROJECT_ID/$REPO/$IMAGE:$TAG .
```

*注意：这将在 Google Cloud 上构建镜像，可能需要几分钟。*

## 4. 部署到 Cloud Run

部署时，我们需要特别注意**内存配置**，因为 Puppeteer/Lighthouse 需要较多内存。

```bash
# 部署服务
gcloud run deploy lighthouse-hq \
    --image $REGION-docker.pkg.dev/$PROJECT_ID/$REPO/$IMAGE:$TAG \
    --region $REGION \
    --platform managed \
    --allow-unauthenticated \
    --memory 2Gi \
    --cpu 1 \
    --timeout 300 \
    --set-env-vars NODE_ENV=production
```

**关键参数说明：**
*   `--memory 2Gi`: **关键！** 分配 2GB 内存以防止浏览器崩溃。如果遇到 OOM 错误，请尝试增加到 `4Gi`。
*   `--timeout 300`: 将请求超时设置为 300秒（5分钟），因为性能检测可能需要较长时间。
*   `--allow-unauthenticated`: 允许公开访问（根据需求可关闭）。

## 5. 配置环境变量

如果在代码中使用了 Supabase 或 Redis，需要设置相应的环境变量。

```bash
gcloud run services update lighthouse-hq \
    --region $REGION \
    --set-env-vars SUPABASE_URL=your_url,SUPABASE_KEY=your_key,REDIS_URL=your_redis_url
```

## 6. 验证部署

部署成功后，命令行会输出一个 URL（例如 `https://lighthouse-hq-xyz.a.run.app`）。
访问该 URL，您应该能看到前端页面。

### 常见问题

**Q: 部署失败，提示内存不足？**
A: 确保在部署命令中指定了 `--memory 2Gi` 或更高。

**Q: Lighthouse 检测超时？**
A: 增加 Cloud Run 的超时设置 `--timeout`，并在代码中调整 Puppeteer 的超时配置。

**Q: 如何查看日志？**
A: 在 Google Cloud Console 中进入 Cloud Run -> Logs，或使用命令：
```bash
gcloud beta run services logs tail lighthouse-hq --project $PROJECT_ID
```
